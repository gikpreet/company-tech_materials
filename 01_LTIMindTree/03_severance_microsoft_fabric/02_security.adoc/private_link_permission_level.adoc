= Microsoft Fabric에서 Private Link 구성 및 관리 권한
:sectnums:
:toc:

Microsoft Fabric에서 Private Link를 구성하고 관리하는 권한은 플랫폼의 **거버넌스**와 **네트워킹** 영역에 속하기 때문에, 일반적인 작업 영역 역할과는 구별되는 **최상위 관리자 권한**이 필수적입니다.

다음은 Microsoft Fabric에서 Private Link와 관련된 세 가지 주요 권한 계층입니다.

== Fabric/태넌트 수준 권한 (설정 및 활성화)

Private Link 기능의 존재 자체와 작동 방식을 제어하는 **최고 권한**입니다.

[cols="2,2,3a", options="header"]
|===
|권한 역할|역할 범위|주요 권한
|Fabric 관리자(Fabric Administrator) +
전역 관리자(Global Administrator)
|조직 전체(태넌트)
|* 관리 포털(Admin 접근 권한)
* Azure Private Link 기능을 활성화/비활성화 할 수 있습니다.
* **공용 인터넷 액세스 차단** 설정을 구성하여 Private Link를 통한 접근만 강제할 수 있습니다.
|===

== Azure RBAC 권한 (인프라 및 연결 관리)

실제로 Azure VNet에 Private Endpoint 리소스를 생성하고 이를 Fabric 서비스에 연결하는데 필요한 권한압니다.

[cols="1,1,3", options="header"]
|===
|권한 역할|역할 범위|주요 권한
|네트워크 기여자(Network Contributor)|Private Endpoint를 만들 VNet 및 서브넷|Private Endpoint 리소스를 VNet에 생성할 수 있습니다.
|리소스 그룹 기여자|Private Endpoint 리소스 그룹|Private Endpoint 리소스에 대한 **전체 관리** 권한을 가집니다.
|대상 서비스 연결 권한|Fabric 용량/태넌트 리소스|Private Endpoint가 Fabric 서비스에 연결되도록 **승인**하거나 **거부**할 수 있습니다. (일반적으로 Fabric 관리자 역할이 이를 포함)
|===

== 사용자 접근 권한(접속 및 사용)

Private Link 구성 후, 최종 사용자가 Fabric에 접근할 때 필요한 권한입니다.

[cols="1,1,3a", options="header"]
|===
|권한 역할|역할 범위|주요 권한
|일반 사용자(모든 Fabric 역할)|작업 영역 및 항목|
* Private Endpoint가 설정된 VNet 또는 연결된 네트워크에 접속할 수 있어야 합니다.
* 네트워크가 Private Link를 통해 올바르게 라우팅되고 **DNS 확인**이 성공해야 Fabric 포털에 접속할 수 있습니다.
|===
